# WiseMed-SmartLink 智慧医疗平台 - 路演展示指南

## 📋 项目概述

### 项目定位
WiseMed-SmartLink 是一个面向基层医疗机构（乡镇卫生院、社区诊所）的**智能化医疗服务平台**，通过 AI 技术赋能基层医疗，解决医疗资源不均、诊疗效率低下、跨语言沟通困难等痛点。

### 核心价值主张
1. **降低门槛**：AI 辅助诊断，帮助基层医生提升诊疗水平
2. **提升效率**：智能化工作流，减少重复性工作，提高接诊量
3. **打破壁垒**：多语言支持，服务国际患者，拓展业务范围
4. **数据驱动**：电子化病历管理，积累医疗数据资产

### 目标用户
- **主要用户**：乡镇卫生院、社区卫生服务中心、民营诊所
- **次要用户**：县级医院、医疗集团、国际医疗机构

## 🎯 核心功能模块

### 1. 临床服务模块

#### 1.1 预约管理
- **功能**：日期选择、预约列表、状态跟踪、快速接诊
- **价值**：规范就诊流程，减少患者等待时间，提高诊室利用率
- **演示要点**：
  - 展示今日预约列表，不同状态（已确认、待确认、已完成）
  - 演示快速接诊流程
  - 强调时间管理和效率提升

#### 1.2 患者登记
- **功能**：详细建档（基本信息、医保信息、紧急联系人）
- **价值**：一次录入，终身使用；支持医保对接，减少重复工作
- **演示要点**：
  - 展示优化后的 UI（大间距、清晰标签、占位符提示）
  - 强调必填项验证和数据完整性
  - 突出医保类型选择（城镇职工、城乡居民、商业保险、自费）

#### 1.3 电子病历 (EMR)
- **功能**：病历列表、详情查看、处方管理、检查申请
- **价值**：无纸化办公，病历永久保存，支持远程会诊和转诊
- **演示要点**：
  - 展示患者病历历史记录
  - 演示处方开具和检查申请流程
  - 强调数据安全和隐私保护

#### 1.4 智能影像
- **功能**：DICOM 影像上传、AI 辅助分析、影像查看
- **价值**：辅助基层医生读片，提高诊断准确率
- **演示要点**：
  - 上传 CT/X 光片
  - 展示 AI 分析结果（病灶标注、诊断建议）
  - 强调 AI 辅助而非替代医生

#### 1.5 全球诊所
- **功能**：实时语音转录（ASR）、医疗术语翻译（NMT）、语音合成（TTS）
- **价值**：打破语言障碍，服务国际患者，拓展业务范围
- **演示要点**：
  - 演示中文问诊 → 英文翻译 → 语音播放
  - 强调医疗术语的专业翻译准确性
  - 突出国际化医疗服务能力

#### 1.6 生命体征监测
- **功能**：体征录入（血压、心率、体温、血氧）、趋势图表
- **价值**：可视化健康数据，辅助慢病管理
- **演示要点**：
  - 展示体征录入界面
  - 演示血压和心率趋势图
  - 强调慢病管理的重要性

### 2. AI 能力中心

#### 2.1 AI 医疗助手（大模型）
- **功能**：类 ChatGPT 的医疗问答界面，支持医学知识查询、症状分析、用药建议
- **价值**：24/7 在线医疗知识库，辅助医生决策，提升患者教育
- **演示要点**：
  - 演示常见问题咨询（如何控制血压？糖尿病饮食建议？）
  - 展示 AI 的专业回答（包含诊断标准、治疗方案、注意事项）
  - 强调 AI 建议仅供参考，不替代医生诊断

#### 2.2 智能导诊
- **功能**：基于症状的科室推荐
- **价值**：减少患者挂错号，提高就诊效率
- **演示要点**：
  - 输入症状（如"头痛伴发热"）
  - 展示 AI 推荐的科室（神经内科/发热门诊）
  - 强调分诊准确性

#### 2.3 用药安全检查
- **功能**：药物相互作用检测
- **价值**：避免用药风险，保障患者安全
- **演示要点**：
  - 选择多种药物（如阿司匹林 + 华法林）
  - 展示严重相互作用警告
  - 强调用药安全的重要性

#### 2.4 模型管理
- **功能**：AI 模型部署、版本管理、性能监控
- **价值**：灵活配置 AI 能力，支持模型迭代升级
- **演示要点**：
  - 展示已部署的模型列表
  - 演示模型部署/卸载操作
  - 强调系统的可扩展性

## 🏗️ 技术架构

### 前端技术栈
- **框架**：Next.js 14 (React 18)
- **样式**：Tailwind CSS + 自定义组件库
- **医学影像**：Cornerstone.js (DICOM 查看器)
- **特点**：
  - 服务端渲染 (SSR) 提升首屏加载速度
  - 响应式设计，支持桌面/平板/移动端
  - 暗黑模式支持

### 后端技术栈
- **框架**：FastAPI (Python)
- **AI 服务**：
  - 影像分析：PyTorch + 医学影像模型
  - 语音识别：Whisper (OpenAI)
  - 机器翻译：自研医疗术语词典 + NMT 模型
  - 语音合成：TTS 引擎
- **数据存储**：
  - 关系型数据库：PostgreSQL
  - 对象存储：MinIO (DICOM 影像)
  - 消息队列：Redpanda (Kafka 兼容)

### 基础设施
- **容器化**：Docker + Docker Compose
- **监控**：Prometheus + Grafana
- **认证**：Keycloak (SSO)
- **影像存储**：Orthanc (PACS 服务器)

## 💡 创新亮点

### 1. AI + 临床深度融合
不是简单的 AI 功能堆砌，而是将 AI 能力深度嵌入临床工作流：
- **智能影像**：影像上传 → AI 分析 → 辅助诊断 → 病历记录
- **全球诊所**：语音问诊 → 实时翻译 → 跨语言沟通 → 病历归档
- **智能导诊**：症状输入 → AI 分析 → 科室推荐 → 快速挂号

### 2. 面向基层医疗的产品设计
- **低学习成本**：直观的 UI，清晰的操作流程
- **高容错性**：AI 建议 + 医生决策，避免过度依赖 AI
- **实用性优先**：聚焦高频场景（预约、登记、开单、影像）

### 3. 开放架构，易于扩展
- **微服务架构**：各服务独立部署，支持灵活扩展
- **模型热插拔**：支持 AI 模型动态加载/卸载
- **标准协议**：DICOM、HL7、FHIR 等医疗标准支持

### 4. 多语言国际化
- **全球诊所模块**：支持中英文实时翻译
- **医疗术语词典**：专业医学术语翻译，准确率高
- **拓展潜力**：可扩展至更多语言（日语、韩语、阿拉伯语等）

## 🚀 部署指南

### 系统要求
- **操作系统**：Linux (Ubuntu 20.04+) / macOS
- **硬件**：
  - CPU：4 核及以上
  - 内存：8GB 及以上（推荐 16GB）
  - 存储：50GB 及以上
- **软件**：
  - Docker 20.10+
  - Docker Compose 2.0+

### 快速部署（5 分钟启动）

```bash
# 1. 克隆代码
git clone https://github.com/mengguiyouziyi/WiseMed-SmartLink.git
cd WiseMed-SmartLink

# 2. 启动所有服务
cd infra/docker
docker compose --profile central up -d

# 3. 等待服务启动（约 2-3 分钟）
docker compose ps

# 4. 访问系统
# 浏览器打开：http://localhost:3001
```

### macOS 部署注意事项
1. **Docker Desktop**：确保已安装并启动 Docker Desktop
2. **资源分配**：建议分配至少 4GB 内存给 Docker
3. **网络模式**：macOS 不支持 `network_mode: host`，需修改 `docker-compose.yml`：
   ```yaml
   # 将 web-service 的 network_mode: "host" 改为
   ports:
     - "3001:3001"
   networks:
     - smartlink-net
   ```
4. **环境变量**：在 `apps/web/package.json` 中确保端口配置正确

## 🎬 演示场景设计

### 场景 1：基层医生的一天（5 分钟）
**角色**：乡镇卫生院医生
**流程**：
1. **早晨 8:00**：打开系统，查看今日预约列表（10 位患者）
2. **接诊第一位患者**：
   - 患者登记（新患者，录入基本信息）
   - 生命体征录入（血压 140/90，心率 85）
   - 开具处方（降压药）
   - 用药安全检查（检测药物相互作用）
3. **接诊第二位患者**：
   - 老患者，查看历史病历
   - 上传 X 光片，AI 辅助分析
   - 根据 AI 建议，开具检查申请
4. **中午休息**：查看生命体征趋势图，关注慢病患者

**演示重点**：
- 完整的临床工作流
- AI 辅助提升效率
- 数据积累和管理

### 场景 2：国际患者就诊（3 分钟）
**角色**：社区诊所医生 + 外国患者
**流程**：
1. 外国患者（英语）描述症状
2. 医生使用"全球诊所"模块：
   - 录音（英文）
   - 实时转录和翻译（中文）
   - 医生用中文回复
   - 系统翻译成英文并语音播放
3. 完成问诊，开具处方

**演示重点**：
- 跨语言沟通能力
- 医疗术语翻译准确性
- 拓展国际业务的可能性

### 场景 3：AI 医疗助手咨询（2 分钟）
**角色**：患者自助查询
**流程**：
1. 患者在候诊时使用 AI 医疗助手
2. 提问："如何控制血压？"
3. AI 给出详细建议（饮食、运动、用药）
4. 患者继续提问："糖尿病患者能吃什么水果？"
5. AI 给出专业回答

**演示重点**：
- AI 的医学知识储备
- 患者教育和自助服务
- 减轻医生负担
