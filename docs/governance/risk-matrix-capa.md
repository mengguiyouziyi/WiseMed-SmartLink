## 2. 风险管理流程
1. **识别**：项目/技术/合规定期评审（至少每季度）。
2. **评估**：采用概率/影响五级评分，形成风险等级矩阵。
3. **计划**：制定控制措施（预防/探测/纠正），设关键指标。
4. **执行**：分配责任人，纳入项目计划。
5. **监控**：Dashboard 跟踪指标；异常触发风险升级。
6. **报告**：风险日志每月更新，重大风险报管理层/投资方。

## 3. CAPA 流程
### 3.1 触发条件
- 合规/监管发现（审核、审计）
- 临床或客户投诉（安全、质量）
- 安全事件（CVE、渗透测试发现）
- 内部指标越线（SLO、模型性能）

### 3.2 步骤
1. **记录 (Log)**：在 CAPA 工单系统（Jira/禅道）创建记录，包含来源、描述、影响。
2. **遏制 (Containment)**：必要时立即采取临时措施（关闭功能、停用模型）。
3. **调查 (Investigation)**：
   - 根因分析（5 Whys/Fishbone）。
   - 收集证据：日志、配置、测试报告。
4. **计划 (Action Plan)**：
   - 纠正措施 (Corrective)：修复现有问题。
   - 预防措施 (Preventive)：避免再次发生。
   - 明确责任人、截止时间、验证方式。
5. **执行 (Implementation)**：跟踪任务完成，必要时代码审查、测试、合规审查。
6. **验证 (Effectiveness Check)**：验证措施有效（测试、指标、审查）。
7. **关闭 (Closure)**：QA/合规签字；归档证据；更新相关文档（SOP、培训）。

## 4. 工具与记录
- CAPA 工单：字段包括编号、类别、来源、影响、风险等级、状态、附件。
- 证据存档：测试报告、日志、邮件记录，存至 Confluence/共享盘。
- 培训回执：若 CAPA 牵涉流程更新，需记录培训名单与测验。

## 5. 报告
- 月度 CAPA 状态报告：打开/关闭数量、平均关闭天数、未关闭P1。
- 季度管理评审：回顾关键风险、CAPA 效果、需新增资源。
