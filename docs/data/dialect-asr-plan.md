# 方言语料采集与模型调优方案

## 1. 目标与指标
- 方言：衡水、保定（可扩展至石家庄/邯郸）
- 场景：门诊问诊、处方宣教、慢病随访、翻译对话。
- 指标：
  - 词准确率 (WER) ≤ 15%
  - 医疗术语召回 ≥ 95%
  - 热词命中率 ≥ 98%
  - 低置信度回退率 < 10%

## 2. 语料策略
| 阶段 | 数据源 | 规模 | 说明 |
| --- | --- | --- | --- |
| 采集 | 实地录音、医患角色扮演、远程随访 | 200h (首年) | 分角色渠道：医生/护士/患者/翻译；签署知情同意。 |
| 增强 | 众包朗读 + 文本合成 | 50h | 使用医疗脚本+方言词典校验。 |
| 公共数据 | AISHELL-3、MagicData-Explore、THCHS-30 | 100h | 提供普通话/方言基线，统一采样率。 |

## 3. 采集流程
1. **前期准备**：
   - 编写剧本（问诊、翻译、宣教、药物提醒），覆盖 ICD-10 高发病种 + 慢病管理。
   - 构建术语词表（药名、剂量、部位、检验指标、宗教禁忌提示）。
   - 审批：伦理委员会、数据最小化、DPA。
2. **现场录制**：
   - 设备：双通道麦克风 + 便携录音箱，48kHz/24bit。
   - 每段 < 5 分钟，标注说话人角色、情绪、场景标签。
3. **标注**：
   - 工具：ELAN / Praat /自研 Web 标注器。
   - 三段式流程：粗转写 → 术语校验 → QA 抽检（≥ 5%）。
   - 输出 JSON（含时间戳、词级置信度、术语标签）。
4. **质量控制**：
   - 双人互审；WER 超过 20% 的样本重录或复核。
   - 方言词典自动比对，缺失词入库并回访采样。
5. **安全与隐私**：
   - 采集端全磁盘加密；上传走 VPN；脱敏处理（姓名/身份证/电话）。
   - 访问日志：采集、标注、训练三类角色分权。

## 4. 模型调优
### 4.1 ASR
- 基线：FunASR/WeNet Conformer + CTC/Attention；使用 `paraformer` 流式模型。
- 适配：
  - 热词注入：基于 Weighted Finite State Transducer (WFST) 或 NeMo热词模块。
  - 多通道降噪：Beamforming + Voice Activity Detection (Silero VAD)。
  - 蒸馏：从大模型（Paraformer-large）蒸馏到边缘版（<120MB）。
- 训练：
  - 80% 方言，20% 普通话混合；SpecAugment + NoiseMix。
  - 花式学习率 (Noam)；FP16 + DDP；用 ESPnet/Wenet recipes。
  - 验证集按场景分桶，输出 WER/TER。

### 4.2 NMT
- 基线：OpenNMT Transformer，预训练医学平行语料（OPUS + 自建）。
- 术语对齐：
  - 通过术语词表构建 `Glossary Constraint`，强制多语言翻译一致。
  - 使用 `copy-mechanism` 保持药品名。
- 评测：BLEU、COMET、术语准确率；低置信度阈值 0.75。

### 4.3 对话管理
- Rasa + 自定义 NLU pipeline，加入医疗意图（过敏、妊娠、宗教禁忌）。
- 敏感意图触发人工转接或安全提示。

## 5. 数据飞轮与迭代
1. 低置信度样本进入人工复核队列；打标签并加入增量数据。
2. 每周小批次再训练（≥ 5h 新语料），灰度发布至 RK3588 设备（10% → 50% → 全量）。
3. 版本管理：模型卡记录数据版本、性能、禁用场景；SBOM 关联。
4. 监控：Grafana Dashboard 展示 WER、术语准确率、低置信度回退率；异常 >2% 触发CAPA。

## 6. 资源需求（首年）
- 人员：数据PM 1、采集执行 2、标注 6（外包）、ASR工程师 2、NLP工程师 1。
- 设备：便携录音6套（2万元）、远程采集App（定制 5万元）。
- 云/算力：A100×4 服务器 1 台（40万元/年），用于模型训练。
