# WiseMed-SmartLink PoC 演示脚本

**版本**: v1.0
**日期**: 2025-11-19
**适用对象**: 利益相关者、技术评审团队、潜在客户

---

## 1. 演示概览

本次演示旨在展示 WiseMed-SmartLink 平台 PoC (概念验证) 阶段的核心能力，重点验证微服务架构的可行性、医学多语言翻译的准确性以及系统的可观测性。

**演示时长**: 约 15-20 分钟

**核心展示点**:
1.  **系统架构**: 基于 Docker Compose 的微服务架构
2.  **医学翻译**: 精确的医学术语识别与翻译
3.  **影像流程**: 模拟的 DICOM 影像处理事件流
4.  **系统监控**: Prometheus + Grafana 全链路监控

---

## 2. 准备工作

在开始演示前，请确保环境已就绪：

1.  **启动环境**:

3.  **打开浏览器页面**:
    - Grafana: http://localhost:3000 (admin/admin)

## 3. 演示流程

### 环节一：系统健康与架构展示 (3 分钟)

**目标**: 展示系统的稳定性和架构设计的专业性。

**操作步骤**:
1.  打开终端，运行健康检查命令：
    ```bash
    curl http://localhost:8001/healthz
    curl http://localhost:8004/healthz
    ```
2.  展示 **Grafana 系统概览仪表板**：
    - 指出 "Services Status" 面板全绿。
    - 展示容器资源使用情况，证明系统负载正常。

**解说词**:
> "大家现在看到的是 WiseMed-SmartLink 的实时监控大屏。我们目前的 PoC 环境由 4 个核心微服务和 7 个基础设施组件组成。通过 Grafana，我们可以实时看到所有服务均处于健康状态（绿色），系统运行稳定。"

### 环节二：医学多语言翻译演示 (5 分钟)

**目标**: 验证平台解决"语言沟通障碍"的能力。

**操作步骤**:
1.  打开终端或 Postman。
2.  **场景 1：基础术语翻译**
    - 发送请求：
      ```bash
      curl -X POST http://localhost:8004/translate \
        -H "Content-Type: application/json" \
        -d '{"text":"患者主诉胸痛，怀疑是心肌梗死"}'
      ```
    - 展示结果：强调 "胸痛" -> "chest pain", "心肌梗死" -> "myocardial infarction"。

3.  **场景 2：复杂混合场景**
    - 发送请求：
      ```bash
      curl -X POST http://localhost:8004/translate \
        -H "Content-Type: application/json" \
        -d '{"text":"既往有高血压病史，近期出现呼吸困难"}'
      ```
    - 展示结果：强调 "高血压" -> "hypertension", "呼吸困难" -> "dyspnea"。

**解说词**:
> "这是我们的核心功能之一——医学智能翻译。不同于普通翻译软件，我们的引擎集成了专业的医学术语库。大家看这个例子，系统不仅翻译了句子，还精准识别出了'心肌梗死'、'高血压'等专业术语，并给出了标准的英文医学对应词。这对于基层医生查阅国际文献或进行远程会诊非常有帮助。"

### 环节三：影像处理工作流 (5 分钟)

**目标**: 展示 AI 辅助诊断的事件驱动架构。

**操作步骤**:
1.  **模拟影像上传** (口头描述 + 架构图展示):
    - 展示架构图中的 `Orthanc -> PACS Gateway -> Redpanda -> AI Inference` 流程。
2.  **验证消息队列**:
    - 在终端运行：
      ```bash
      docker exec redpanda rpk topic list
      ```
    - 指出 `exam.created` 主题的存在。
3.  **查看 AI 服务日志**:
    - 运行：
      ```bash
      docker logs ai-infer-service --tail 10
      ```
    - 说明服务正在监听消息，准备进行推理。

**解说词**:
> "在影像诊断方面，我们采用了先进的事件驱动架构。当基层医院拍摄 X 光片后，影像会自动上传到我们的 PACS 网关，并通过消息队列触发 AI 推理服务。虽然目前 PoC 阶段我们使用的是模拟模型，但这套架构已经完全打通，可以无缝接入真实的 MONAI AI 模型。"

### 环节四：总结与问答 (2-5 分钟)

**总结要点**:
- PoC 目标已达成：架构验证通过。
- 核心功能可用：翻译准确，服务稳定。
- 下一步计划：引入真实 AI 模型，开展试点部署。

## 4. 常见问题 (FAQ)

**Q: 翻译服务支持语音输入吗？**
A: 架构上已经支持（集成了 Whisper 模型），PoC 阶段为了演示稳定性，目前主要展示文本输入接口。语音接口 `/asr/transcribe` 已经就绪。

**Q: 系统安全性如何？**
A: 我们集成了 Keycloak 进行统一身份认证，所有服务间通信都在 Docker 内部网络进行。下一步我们会启用全链路 HTTPS。

**Q: 这套系统能部署在医院本地吗？**
A: 可以。我们的微服务架构非常灵活，支持云端部署，也支持在医院本地服务器甚至高性能边缘设备（如 RK3588）上部署。
